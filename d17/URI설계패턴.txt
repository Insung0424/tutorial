uri
인터넷에서 특정자원을 나타내는 주소값 (유일함)
응답은 달라질 수 있다. 
url
인터넷에서의 자원,특정 파일이 어디에위치하는지 식별주소
 url 은 uri의 하위개념

uri 설계원칙(RFC-3986 에 정의되있음)
 / 는 계층관계를 나타내는데 사용, 마지막에 는 포함하지 않음
 - 는 가독성을 높이는데 사용, _은 사용하지 않음 
 경로에는 소문자가 적합
 파일확장자는 포함하지 않음
 프로그래밍언어 의존적인 확장자 사용하지 않음
 구현에 의존적인 경로사용하지 않음
 세션아이디 를 포함하지 않음
 메소드명을 이용하지 않는다
 명사는 복수형을 사용 컬렉션에대한표현은 복수로 사용
 컨트롤러 이름으로는 동사나 동사구로 사용
 경로부분중 변하는부분은 유일한값으로 사용
 CRUD기능을 나타내는것은 사용하지않음
 쿼리파라미터디자인-쿼리부분으로 컬렉션결과에대해 필터링0
 API에 서브도메인은 일관성있게 사용
 클라이언트개발자포탈 서브도메인 일관성있게 사용(실사용전 테스트용)
